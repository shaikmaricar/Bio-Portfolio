01
— Foundations

Install Python (pick a path)

You have two clean options. Both work, both are well-supported. Miniconda is the gentler path if you plan to juggle many environments, while a direct Python install is the most minimal route — just the language, nothing else.

Option A · Recommended

Miniconda

A small installer that ships conda, Python, and not much else. Great for managing many projects.

docs.conda.io/miniconda →
Option B · Minimal

Python.org installer

The official Python distribution. Smallest footprint, native venv support, no extras.

python.org/downloads →

After installing, confirm that Python is on your PATH:

Terminal — verify install 
~/projects $ python --version
Python 3.12.4

~/projects $ pip --version
pip 24.1.2 from C:\Users\you\AppData\Local\Programs\Python\...
!
Windows users: on the Python installer, tick "Add python.exe to PATH" before clicking Install Now. It saves you a config headache later.
02
— Isolation

Create a virtual environment

A virtual environment is a sandboxed Python copy that keeps this project's dependencies from leaking into others. Pick a folder, name your environment anything you like (here we use llama-env), and run:

Terminal — create venv 
# 1. create the environment
~/projects $ python -m venv llama-env

# 2. activate it (Windows / cmd.exe)
~/projects $ llama-env\Scripts\activate.bat

# 3. activate it (PowerShell)
~/projects $ llama-env\Scripts\Activate.ps1

# 4. activate it (macOS / Linux)
~/projects $ source llama-env/bin/activate

(llama-env) ~/projects $

When the prompt prefix changes to (llama-env), the environment is active and any pip install you run from now on lands inside it — not your global Python.

i
Naming tip: short, lowercase, no spaces. Common conventions are .venv, env, or a project-specific name.

— Configuration

Select the Python interpreter

With your workspace open, hit Ctrl + Shift + P (or ⌘ + Shift + P on macOS) to open the Command Palette. Start typing Python: Select Interpreter and pick the one inside your llama-env folder.

06
— Notebook

Create a new Jupyter notebook

Open the Command Palette again (Ctrl + Shift + P), and this time type Create: New Jupyter Notebook. VS Code will spin up an empty notebook in a new tab.

07
— Persistence

Save it as first-test.ipynb

Press Ctrl + S and save the file inside your workspace folder. Use the filename first-test.ipynb — the .ipynb extension is what makes VS Code (and the broader Jupyter ecosystem) recognise it as a notebook.

08
— Hello, model

Write & run your first prompt

Click into the first cell of the notebook and paste in the four lines below. Then hit Shift + Enter to execute it. The first run may take a few seconds as Ollama warms up the model.

first-test.ipynb — cell 1 of 1 ● llama-env (Python 3.12.4)
In [1]:
from llama_index.llms.ollama import Ollama

llm = Ollama(
    model="llama3.2:latest",
    temperature=0.2,
    request_timeout=30,
    max_retries=3,
    max_tokens=2048,
)

response = llm.complete("What is the capital of France?")
print(response)
Out[1]:
The capital of France is Paris. It is one of the most visited cities in
the world, known for landmarks such as the Eiffel Tower, the Louvre
Museum, and Notre-Dame Cathedral.

That output is being generated entirely on your machine — no network call to a hosted model, no token usage charged anywhere. You've just wired up your first local LLM pipeline.

i
What each parameter does: temperature controls randomness (lower = more deterministic), request_timeout caps how long to wait, max_retries handles transient failures, and max_tokens bounds the response length.
— You shipped it

The complete recipe, distilled.

Eight steps, one working local LLM. From here, the obvious next moves: swap models (try mistral or qwen2.5), add a VectorStoreIndex over your own documents, or stream tokens with llm.stream_complete().
